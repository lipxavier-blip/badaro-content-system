# 7. Content Governance — Governança de Conteúdo

## Fluxo de Criação

```
Estratégia → Criação → Revisão Editorial → Aprovação → Publicação → Atualização
```

### 1. Estratégia
- Definir objetivo do conteúdo (o que queremos que o leitor faça ou pense depois)
- Identificar o canal e o tom adequado (ver `4-voice-and-tone.md`)
- Identificar o padrão de estrutura a usar (ver `6-content-patterns.md`)

### 2. Criação
- Seguir os princípios de foundations (`1-foundations.md`)
- Aplicar a identidade verbal (`2-verbal-identity.md`)
- Usar o messaging framework como base (`3-messaging-framework.md`)

### 3. Revisão Editorial
Checklist antes de publicar:
- [ ] O tom está alinhado com o contexto? (ver matriz em `4-voice-and-tone.md`)
- [ ] Sem buzzwords? (ver lista em `5-editorial-style-guide.md`)
- [ ] Voz ativa predominante?
- [ ] Frases curtas e diretas?
- [ ] Terminologia correta? (ver glossário em `5-editorial-style-guide.md`)
- [ ] A sigla foi apresentada na primeira menção?
- [ ] O conteúdo orienta uma decisão ou ação?

### 4. Aprovação
- Conteúdo institucional (site, proposta): aprovação do responsável pela marca
- Conteúdo de redes sociais: revisão editorial + aprovação rápida
- Artigos e newsletters: revisão editorial suficiente

### 5. Publicação
- Registrar o conteúdo publicado no canal correto
- Guardar versão final no repositório do projeto

### 6. Atualização
- Revisar conteúdos institucionais a cada 6 meses
- Atualizar glossário e anti-voz sempre que novos padrões forem identificados
- Registrar aprendizados em `skills/learn.md`

## Responsabilidades

| Papel | Responsabilidade |
|---|---|
| Estrategista de conteúdo | Define objetivo, canal e estrutura |
| Redator / content designer | Cria seguindo os padrões do sistema |
| Revisor editorial | Aplica o checklist antes da publicação |
| Responsável pela marca | Aprova conteúdo institucional |

---

## Atualização deste Sistema

O Content System é um documento vivo. Deve ser atualizado quando:
- Um novo padrão de linguagem for identificado e validado
- Um erro recorrente for detectado nas revisões
- O posicionamento da marca evoluir
- Novos canais forem adicionados à operação

Registre qualquer aprendizado relevante em `skills/learn.md` antes de editar os documentos centrais.
